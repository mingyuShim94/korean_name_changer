# Overview

## 프로젝트명: MyKoreanName (한국식 이름 생성기)

**설명:** 외국인의 이름을 한국식 이름으로 의미 있게 재해석해주는 시적 네이밍 웹서비스입니다. 이 서비스는 외국 이름을 가진 사용자들이 자신의 이름에 담긴 의미와 정서를 한국 문화의 아름다움과 연결하여 새로운 정체성을 탐색할 수 있도록 돕는 것을 목표로 합니다. 사용자가 자신의 외국 이름을 입력하면, 서비스는 그 이름의 어원, 상징, 느낌 등을 분석하여 한국적인 감성과 전통을 담은 아름다운 한국식 이름(성 포함)을 생성하고, 그 의미와 시적 해석을 함께 제공합니다.

**문제 해결:**

- 외국인들이 한국 문화에 대한 관심이 높아짐에 따라 한국식 이름을 갖고 싶어하는 수요가 있습니다.
- 단순히 발음만 비슷한 이름이 아닌, 자신의 본래 이름과 의미적으로 연결되는 깊이 있는 한국 이름을 찾기 어렵습니다.
- 문화적 배경과 언어적 뉘앙스를 이해하고 시적인 아름다움을 담아 이름을 짓는 것은 전문적인 지식을 필요로 합니다.

**대상 사용자:**

- 한국 문화에 관심 있는 외국인
- 한국인 친구나 동료와 더 깊은 유대감을 형성하고 싶은 외국인
- 자신의 아이에게 특별한 의미를 담은 한국식 이름을 지어주고 싶은 다문화 가정 부모
- 새로운 캐릭터나 필명을 구상하는 창작자

**가치 제안:**

- 의미 있는 연결: 사용자의 본래 이름과 의미적으로 연결되는 아름다운 한국식 이름을 제공합니다.
- 시적 감성: 단순한 번역을 넘어 시적이고 감성적인 해석을 더해 이름에 특별한 가치를 부여합니다.
- 문화적 이해: 한국의 성씨와 이름에 담긴 문화적 배경과 의미를 함께 전달하여 한국 문화에 대한 이해를 높입니다.
- 간편한 사용성: 직관적인 인터페이스를 통해 누구나 쉽게 자신만의 특별한 한국 이름을 만들 수 있습니다.
- 다양한 옵션: 성별 뉘앙스(남성형, 여성형, 중성형)와 이름 스타일(한자 기반, 순우리말) 선택 가능합니다.

# Core Features

## 가. 이름 입력 및 옵션 선택 UI

- 사용자가 외국 이름(예: "Isabella Rossellini") 또는 이름만 입력할 수 있는 단일 입력창 제공
- 성별 뉘앙스 선택 옵션:
  - 남성형(Masculine): 남성적인 느낌의 이름 생성
  - 여성형(Feminine): 여성적인 느낌의 이름 생성
  - 중성형(Neutral): 성별에 구애받지 않는 이름 생성
- 이름 스타일 선택 옵션:
  - 한자 기반(Hanja-based): 전통적인 한자를 기반으로 한 이름
  - 순우리말(Pure Korean): 한자 없이 순수한 한국어 단어를 활용한 이름
- 무료/프리미엄 서비스 선택 탭 (베타 테스트 중에는 모두 무료 제공)
- "Generate Korean Name" 버튼으로 이름 생성 시작

## 나. 무료/프리미엄 모드

- 무료 모드: 기본적인 한국 이름 생성 및 해석 제공
- 프리미엄 모드: 더 자세하고 심층적인 이름 해석, 문화적 의미, 발음 특성, 삶의 방향성 등 추가 정보 제공
- 베타 테스트 기간 동안에는 프리미엄 기능도 무료로 제공

# User Experience

## 가. 사용자 페르소나

- 알렉스 (20대 후반, 미국인, K-팝 팬): K-팝과 한국 드라마를 통해 한국 문화에 깊은 관심, 자신을 더 잘 나타낼 수 있는 한국식 이름을 갖고 싶어 함
- 소피아 (30대 초반, 프랑스인, 디자이너): 한국 전통 공예와 디자인에 매료, 예술적 감각과 어울리는 세련된 한국 이름을 원함

## 나. 주요 사용자 흐름

1. 방문 및 이름 입력
   - 홈페이지 방문
   - 무료/프리미엄 모드 선택

   - 생성된 한국 이름 확인

# Technical Architecture

## 가. 시스템 구성 요소

- 프론트엔드: Next.js (App Router), Shadcn UI, Tailwind CSS
- AI 모델 API: Google Gemini API (genai-2.5-flash-preview-04-17)
- 분석: Google Analytics 통합
- Edge Runtime 최적화: Cloudflare Pages 배포

## 나. 데이터 모델

- 기본 데이터 모델: Core Features > 결과 출력 영역 참고
- 프리미엄 데이터 모델: 추가 해석 필드 포함

## 다. API 및 통합

- Next.js Server Actions에서 Gemini API 직접 호출
- 폴백 메커니즘으로 API Routes 활용
- 시스템 프롬프트를 통한 정확한 출력 형식 제어
- Edge Runtime을 통한 최적화된 성능

## 라. 인프라 요구사항

- Cloudflare Pages 또는 Vercel을 통한 호스팅
- Edge Functions 지원을 위한 Cloudflare 설정
- 커스텀 도메인 구성 (선택사항)

- 성별 뉘앙스에 따른 다이나믹 시스템 인스트럭션
- 이름 스타일(한자/순우리말)에 따른 출력 형식 조정

# Risks and Mitigations

## 가. 기술적 도전 과제

- AI 모델의 일관성/품질: 프롬프트 엔지니어링 최적화, 출력 형식 표준화
- Edge Runtime 제약: 적절한 폴백 메커니즘 구현
- 접속자 폭증 시 API 속도/비용: 캐싱 전략 및 속도 제한 구현

## 나. MVP 범위 설정

- 핵심 가치 집중: 필수 기능 우선 구현 후 점진적 개선
- 무료/프리미엄 전략: 베타 테스트 동안 모든 기능 무료 제공하여 피드백 수집

## 다. 리소스 제약

- 익숙한 기술 우선 사용: Next.js, Tailwind, Shadcn UI 활용
- Edge 최적화: 서버리스 아키텍처로 운영 비용 최소화
- API 비용 관리: 적절한 모델 선택 및 토큰 최적화

## 라. 테스트 참고 사항

- 다양한 입력값: 다양한 언어 및 이름 형식 테스트
- 오류 처리: API 실패, 잘못된 입력 처리
- 모바일 최적화: 다양한 화면 크기 및 터치 인터페이스 테스트
