# Overview

## 프로젝트명: Korean Name Changer

**설명:** 외국인의 이름을 한국식 이름으로 의미 있게 재해석해주는 시적 네이밍 웹서비스입니다. 이 서비스는 외국 이름을 가진 사용자들이 자신의 이름에 담긴 의미와 정서를 한국 문화의 아름다움과 연결하여 새로운 정체성을 탐색할 수 있도록 돕는 것을 목표로 합니다. 사용자가 자신의 외국 이름을 입력하면, 서비스는 그 이름의 어원, 상징, 느낌 등을 분석하여 한국적인 감성과 전통을 담은 아름다운 한국식 이름(성 포함)을 생성하고, 그 의미와 시적 해석을 함께 제공합니다.

**문제 해결:**

- 외국인들이 한국 문화에 대한 관심이 높아짐에 따라 한국식 이름을 갖고 싶어하는 수요가 있습니다.
- 단순히 발음만 비슷한 이름이 아닌, 자신의 본래 이름과 의미적으로 연결되는 깊이 있는 한국 이름을 찾기 어렵습니다.
- 문화적 배경과 언어적 뉘앙스를 이해하고 시적인 아름다움을 담아 이름을 짓는 것은 전문적인 지식을 필요로 합니다.

**대상 사용자:**

- 한국 문화에 관심 있는 외국인
- 한국인 친구나 동료와 더 깊은 유대감을 형성하고 싶은 외국인
- 자신의 아이에게 특별한 의미를 담은 한국식 이름을 지어주고 싶은 다문화 가정 부모
- 새로운 캐릭터나 필명을 구상하는 창작자

**가치 제안:**

- 의미 있는 연결: 사용자의 본래 이름과 의미적으로 연결되는 아름다운 한국식 이름을 제공합니다.
- 시적 감성: 단순한 번역을 넘어 시적이고 감성적인 해석을 더해 이름에 특별한 가치를 부여합니다.
- 문화적 이해: 한국의 성씨와 이름에 담긴 문화적 배경과 의미를 함께 전달하여 한국 문화에 대한 이해를 높입니다.
- 간편한 사용성: 직관적인 인터페이스를 통해 누구나 쉽게 자신만의 특별한 한국 이름을 만들 수 있습니다.

# Core Features

## 가. 이름 입력 UI

- 사용자가 외국 이름(예: "Isabella Rossellini") 또는 이름만 입력할 수 있는 단일 입력창 제공
- 텍스트 입력 필드와 "한국 이름 생성" 버튼으로 구성, 반응형 디자인 적용

- AI가 생성한 한국식 이름과 관련 정보를 시각적으로 명확하게 보여줌

## 다. 결과 히스토리 (선택사항 - MVP 이후 고려)

- 세션 내에서 생성된 이름들을 기록하고 목록 형태로 보여줌 (세션 스토리지 활용)

# User Experience

## 가. 사용자 페르소나

- 알렉스 (20대 후반, 미국인, K-팝 팬): K-팝과 한국 드라마를 통해 한국 문화에 깊은 관심, 자신을 더 잘 나타낼 수 있는 한국식 이름을 갖고 싶어 함
- 소피아 (30대 초반, 프랑스인, 디자이너): 한국 전통 공예와 디자인에 매료, 예술적 감각과 어울리는 세련된 한국 이름을 원함

## 나. 주요 사용자 흐름

1. 방문 및 이름 입력
2. 결과 확인
3. 추가 행동(다른 이름 시도, 향후 기능: 저장/공유 등)

## 다. UI/UX 고려사항

- 직관적이고 미적인 디자인, 반응형 웹, 명확한 피드백, 정보의 가독성(Shadcn UI 활용)

# Technical Architecture

## 가. 시스템 구성 요소

- 프론트엔드: Next.js (App Router), Shadcn UI, Tailwind CSS
- AI 모델 API: Google Gemini API
- 백엔드(MVP 이후): Supabase (사용자 인증, 기록 저장 등)

## 나. 데이터 모델

- AI가 반환할 JSON 구조는 Core Features > 결과 출력 영역 참고

## 다. API 및 통합

- Next.js API Route(또는 Server Actions)에서 Gemini API 호출 권장
- API 요청 시 사용자 입력과 시스템 프롬프트를 함께 전달
- 응답(JSON) 파싱 후 UI에 표시

## 라. 인프라 요구사항

- Vercel/Netlify 등 Next.js 호스팅 플랫폼, 도메인 등록

## 마. AI 프롬프트 구조

- 시스템 인스트럭션, 입력/출력 규칙, 예시 등은 기존 PRD의 AI 프롬프트 섹션 참고(내용 유지)

# Development Roadmap

## 가. MVP 요구사항

1. 이름 입력 UI 구현
2. Gemini API 연동 또는 목업 데이터 준비
3. 결과 출력 UI 구현
4. 기본 레이아웃 및 스타일링
5. 테스트 및 배포 준비

## 나. 향후 기능(MVP 이후)

- 사용자 인증, 즐겨찾기, 이름 인증서 이미지, 고급 검색/필터, 다국어 지원, 다양한 이름 생성 옵션, 성씨 선택 등

## 다. 완료 기준

- 외국 이름 입력 → 생성 → 결과 표시(구조화된 정보)
- Shadcn UI 적용, 반응형 웹, API 연동 또는 목업 데이터, JSON 구조 유효성 검사 및 오류 처리

# Logical Dependency Chain

1. 기본 프로젝트 설정(Next.js, Shadcn UI, Tailwind CSS)
2. 핵심 UI 요소 구현(입력창, 버튼, 결과 표시)
3. AI 연동 또는 목업 데이터 준비
4. 데이터 바인딩 및 결과 표시
5. 스타일링 및 반응형 디자인
6. 테스트 및 오류 처리
7. 배포

## 가. 기술적 도전 과제

- AI 모델의 일관성/품질, 한자 및 문화적 정확성
- 완화: 프롬프트 엔지니어링, 결과 후처리, 재시도 옵션, 한자 풀 제한, 사용자 피드백, 전문가 검토 등

## 나. MVP 범위 설정

## 다. 리소스 제약

- 익숙한 기술 우선 사용, MVP 단순화, 오픈소스/커뮤니티 활용
